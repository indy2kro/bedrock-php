    Bedrock PHP — Compliance Declaration¶

    Purpose¶

    This document records a project’s explicit adoption of Bedrock PHP and defines the scope of its compliance.

    It exists to eliminate ambiguity.

    If this document does not exist, the project is not compliant.

    Bedrock PHP Adoption¶

    • Adopted version: Bedrock PHP v1.0
    • Adoption date: YYYY-MM-DD

    By completing this document, the project affirms that it:

    • follows all normative rules in the specified version
    • treats declared defaults as binding
    • understands and accepts the stop rules

    List any intentional deviations from Bedrock PHP v1.0.

    For each deviation, include:

    • Chapter / Rule:

    • Reason:

    • Review date:

    Example:

    • Chapter 5 — Rule 4 (Minimal Infrastructure Surface Area)
    • Using managed message queue due to external integration requirements
    • Required by upstream vendor contract
    • Added operational complexity
    • Review by 2026-01-01

    Review Cadence¶

    This declaration will be reviewed:

    • [ ] quarterly
    • [ ] biannually
    • [ ] annually

    Review includes: - validating continued fit - reassessing deviations - confirming ownership

    Exit Criteria Acknowledgement¶

    The project acknowledges that it must exit Bedrock PHP if:

    • deviations accumulate across multiple chapters
    • core constraints block required outcomes
    • the system no longer operates within the declared defaults

    Exit will be documented per Chapter 11.
